# {システム名}：要件

## 1. 概要

- 本書は {システム名} の要件を、チケット形式（ID・受け入れ条件）で管理する。
- 反映状況（どの文書まで届いたか）は要件トレーサビリティ（10）の§2マトリクスで一元管理する。

## 2. 共通定義

### 2-1. ID採番

- 機能要件 `R-{連番}`／非機能要件 `NFR-{連番}`。一度振ったIDは不変・廃止しても欠番にしない。

### 2-2. 区分・優先度

- 区分：機能要件 / 非機能要件 ／ 優先度：Must / Should / Could / Won't

### 2-3. タグ定義

| タグ軸 | 値（語彙） | 必須 | 説明 |
| --- | --- | --- | --- |
| 区分 | 機能要件 / 非機能要件 | 必須 | 2-2 と一致 |
| 非機能種別 | 性能 / セキュリティ / 可用性 / 運用性 / 保守性 / ユーザビリティ / 互換性 | 非機能のみ必須 |  |
| 領域 | （プロジェクトで定義） | 任意 | 業務領域 |

### 2-4. ステータス定義

- ライフサイクル：起票 → 承認済 → 対応中 → 検証中 → 完了（例外：保留 / 却下 / 廃止＝取り下げコンテキスト必須）
- 反映状況：層ごとに 未 / 反映中 / 済（対象外の層は −）
