## 5. 変更フロー（本プロジェクト適用）

1. 変更は要件チケット（03）起点。誤記修正・リファクタは「要件外の改訂ログ」（10 §4）へ
2. 上流→下流の順に反映（用語集 → 情報定義 → 機能仕様 → API仕様 → 実装 → 品質保証。非機能はアーキへ）
3. 反映のたびにトレーサビリティ（10）の §2 マトリクス＋§3 タイムラインを更新（AIスキル：change-propagate）
