# 要件 — 図書貸出システム

> 本書は[要件フォーマット](../formats/03-requirement.md)に沿った実インスタンス。
> 要件ID（R-xxx）は**トレーサビリティ連鎖の起点**。反映状況の時系列は[要件トレーサビリティ](10-traceability.md)で一元管理する。

## 共通定義

- **ID採番**：機能要件 `R-{連番}`／非機能要件 `NFR-{連番}`。一度振ったIDは変えない・廃止しても欠番にしない。
- **優先度**：Must / Should / Could / Won't
- **ライフサイクル**：起票 → 承認済 → 対応中 → 検証中 → 完了（例外：保留・却下・廃止＝取り下げコンテキスト必須）
- **反映状況**：[トレーサビリティ](10-traceability.md) §2 のマトリクスで一元管理（状態は 未／反映中／済・対象外は −）。

### タグ定義

| タグ | 意味 |
| --- | --- |
| `#貸出` | 貸出・返却の中核業務 |
| `#検索` | 蔵書の照会 |
| `#延滞` | 期限超過の検知・可視化 |
| `#ルール` | 業務ルール（上限・期間など） |
| `#非機能` | 性能・可用性など |

## R-001 蔵書をISBNで検索する `#検索`

- **要望**：司書がISBNで書誌を1件特定し、在架/貸出中の状況を確認したい。
- **受け入れ条件**
  - AC-1：13桁のISBNで書誌を1件特定できる。
  - AC-2：該当なしのとき「見つかりません」を返す。
  - AC-3：ISBNの無い古書はタイトル部分一致で検索できる。
- **コンテキスト**：窓口の問い合わせ対応の即応性向上（管理者要望・司書ヒアリング）。
- **反映状況** → [トレーサビリティ](10-traceability.md) 参照（情報定義→機能仕様→API仕様→実装→品質保証）。

## R-002 貸出を登録する `#貸出` `#ルール`

- **要望**：利用者と蔵書を指定して貸出を登録したい。
- **受け入れ条件**
  - AC-1：在架（`AVAILABLE`）の蔵書のみ貸出できる。貸出中の蔵書は二重貸出できない。
  - AC-2：返却期限＝貸出日＋会員区分の貸出期間（一般14日／学生28日）で自動計算する。
  - AC-3：貸出登録後、蔵書状態は `ON_LOAN`、貸出状態は `ON_LOAN` になる。
- **コンテキスト**：紙台帳の二重貸出ミスを構造的に無くす。

## R-003 返却を登録する `#貸出`

- **要望**：貸出中の本を返却登録し、蔵書を貸出可能に戻したい。
- **受け入れ条件**
  - AC-1：返却登録で蔵書状態が `AVAILABLE`（予約があれば `HELD`）に戻る。
  - AC-2：返却日＞返却期限のとき、延滞日数を記録する。
  - AC-3：貸出状態は `RETURNED` になる。

## R-004 延滞している貸出を一覧する `#延滞`

- **要望**：返却期限を過ぎた貸出を、利用者・延滞日数つきで一覧したい。
- **受け入れ条件**
  - AC-1：返却期限＜当日 かつ 未返却 の貸出を抽出する。
  - AC-2：利用者氏名・書名・延滞日数を表示する。

## R-005 会員区分ごとの貸出上限を超えたら拒否する `#ルール`

- **要望**：上限冊数（一般5／学生10）を超える貸出を拒否したい。
- **受け入れ条件**
  - AC-1：貸出中冊数が上限に達している利用者の新規貸出を拒否する。
  - AC-2：拒否理由（上限超過・現在冊数・上限）を返す。
- **コンテキスト**：R-002 貸出登録の一部として組み込む（独立画面は持たない）。

## R-006 蔵書を予約する `#貸出` — **保留**

- **要望**：貸出中の書誌をタイトル単位で予約（取り置き）したい。
- **ステータス**：保留（起票 2026-07-07）
- **取り下げコンテキスト**：初期リリース（v1.0）は窓口業務の紙台帳置き換えを優先し、予約は v1.1 以降で再検討する（判断：図書館管理者・2026-07-07）。[構想](01-concept.md)§4 のスコープも「含まない（初期リリース）」へ移動済み。
- **先行定義の扱い**：Holdエンティティ（[情報定義](05-info-def.md) 1-5）・`HoldStatus`（[用語集](02-glossary.md)§3）・返却時のHELD分岐は、将来のために**消さず残置**（不変ルール）。

## R-007 延滞を通知する `#延滞`

- **要望**：延滞検出ジョブ（B-001）の結果を、延滞中の利用者へメールで通知したい。
- **受け入れ条件**
  - AC-1：B-001 完了後、延滞中の利用者のメールへ通知を送信する。
  - AC-2：送信失敗はリトライし、失敗一覧を運用が確認できる（[ランブック](12-ops-runbook.md)§2）。
- **コンテキスト**：機能仕様（B-001）・アーキ（通知ロール）・ランブックに実体が先行していたため、**後追いで起票して孤児を解消**（2026-07-07・consistency-check の指摘 C）。
- **ステータス**：対応中

## R-008 蔵書を除籍する `#ルール`

- **要望**：破損・紛失した蔵書を除籍（貸出対象から除外）したい。
- **受け入れ条件**
  - AC-1：在架（`AVAILABLE`）の蔵書を除籍でき、状態が `DISCARDED` になる。
  - AC-2：貸出中の蔵書は除籍できない。
- **コンテキスト**：状態遷移（在架→除籍）が先行定義されていたため後追いで起票（2026-07-07・consistency-check の指摘 B）。機能設計はこれから。
- **ステータス**：承認済
