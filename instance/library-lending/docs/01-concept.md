# 構想／コンセプト — 図書貸出システム

> 本書は[構想フォーマット](../formats/01-concept.md)に沿った実インスタンス。用語は[用語集](02-glossary.md)を正とする。

## 1. これは何か（一文）

公共図書館の蔵書を、利用者が会員区分ごとの上限まで借り・返し・予約できるようにする貸出管理システム。

## 2. 背景・課題

- 貸出/返却を紙台帳で管理しており、二重貸出・延滞把握が属人的でミスが出る。
- 蔵書の所在（在架／貸出中）が即時に分からず、問い合わせ対応に時間がかかる。
- 会員区分ごとの上限冊数の運用が窓口任せで、超過貸出が発生している。

## 3. あるべき姿（North Star）

- 蔵書の状態が常に一意に確定し、二重貸出が構造的に起きない。
- 貸出・返却・延滞・上限が**ルールとして仕組み化**され、窓口の判断に依存しない。
- 延滞は自動検出され、一覧・通知につながる。

## 4. スコープ

| 含む | 含まない（初期リリース） |
| --- | --- |
| 蔵書検索・貸出・返却・延滞一覧・延滞通知・上限管理 | **予約（R-006 保留・v1.1候補）**・会計（延滞金徴収）・蔵書発注・利用者のセルフ貸出端末 |

> 改訂（2026-07-07）：予約を初期リリースから外した（窓口業務の置き換えを優先。[要件](03-requirement.md) R-006 の取り下げコンテキスト参照）。

## 5. 主要ステークホルダ

- 利用者（一般／学生）：借りる人。
- 司書（窓口）：貸出・返却を操作する運用者。
- 図書館管理者：蔵書・利用者マスタと運用ルールを管理。

## 6. 成功の測り方（North Star Metric）

- 二重貸出の発生：**0件**（構造的に防止）。
- 蔵書状態の照会応答：**1秒以内**（[要件](03-requirement.md) の非機能 NFR-01）。
- 延滞の検出漏れ：**0件**（毎日の自動検出）。
