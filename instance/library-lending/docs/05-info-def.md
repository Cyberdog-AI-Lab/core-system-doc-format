## 2. リレーション

- 書誌 1 — N 蔵書（1タイトルに複数コピー）
- 蔵書 1 — N 貸出（履歴）／ただし**未返却は同時に1件まで**（下記 §3）
- 利用者 1 — N 貸出／1 — N 予約
- 書誌 1 — N 予約

## 3. 業務ルール・バリデーション（整合性担保）

### 3-1. 業務ルール

- **返却期限**：`dueOn = loanedOn + loanPeriodDays(memberType)`（[用語集](02-glossary.md) §4）。
- **延滞**：`returnedOn` が空 かつ `dueOn < 当日` → 延滞。

### 3-2. バリデーション

#### V-1 ISBN形式（単項目）
- 対象：書誌.`isbn` ／ 条件：13桁数字・チェックディジット（[用語集](02-glossary.md) §2） ／ タイミング：登録時 ／ 違反時：拒否 ／ 担保場所：アプリ入力チェック

#### V-2 期限の整合（項目間）
- 対象：貸出.`loanedOn`・`dueOn` ／ 条件：`loanedOn ≤ dueOn` ／ 登録時 ／ 拒否 ／ アプリ

#### V-3 二重貸出禁止（エンティティ間）
- 対象：貸出（`copyId` × 未返却） ／ 条件：未返却の貸出は蔵書ごとに同時1件まで ／ 登録時 ／ 拒否 ／ **DB一意制約**（§4）

#### V-4 貸出上限（エンティティ間）
- 対象：利用者 × 貸出 ／ 条件：未返却件数 ≤ `loanLimit`（[用語集](02-glossary.md) §4） ／ 登録時 ／ 拒否（現在冊数・上限を返す） ／ アプリ集計判定 → [機能仕様](06-func-spec.md) F-002・[要件](03-requirement.md) R-005

## 4. 整合性の担保場所

- 「蔵書ID＋未返却」を**一意制約**として物理に持たせ、二重貸出をDBで防止（[アーキ](04-architecture.md) NFR-02／ADR-001）。
- 貸出登録は「在架確認→貸出INSERT→蔵書状態更新」を単一トランザクションで原子的に行う。

## 5. 物理方針（初期採用＝RDB）

- 主キーは代理キー（自動採番）。FKで参照整合性を担保。
- インデックス：`書誌.isbn`（検索）、`蔵書.status`、`貸出(copyId) WHERE returnedOn IS NULL`（一意・未返却）。
