# 文書体系ダッシュボード

`formats/`（または各プロジェクトの `docs/`）の12文書を**全体俯瞰**するための Web ダッシュボード。
ビルド不要の単一HTML。`formats/*.md` を**ブラウザから動的に fetch** して描画するので、
文書を編集して再読込すれば内容が反映される（＝内容は可変）。

## 3つのビュー

| ビュー | 内容 |
|--------|------|
| 俯瞰 | 整合性モデル（SVG・ボックスをクリックで本文へ）＋12文書カード＋貫通線（3カテゴリ7本） |
| トレーサビリティ | **要件チケットの進捗タイムライン**（各チケットが 情報定義→機能仕様→API仕様→アーキ→実装→品質保証 のどこまで反映されたか・反映日つき。対象外の層は −）。上部に**状態別サマリのチップ（クリックで絞り込み）とID/タイトル検索**、下に**自動チェック**（済の日付漏れ・層の順序逆転・ID重複）。さらに貫通線、補助として文書間の参照マトリクス |
| ドキュメント | 文書リスト＋Markdown描画＋目次＋参照チップ。**文書内の .md リンクはビューア内で開く**（体系外のパスは新しいタブ） |

> 進捗タイムラインは、要件トレーサビリティ文書の「トレーサビリティ・マトリクス」（§2）を
> そのまま読み込んで描画する。セルの書き方はフォーマット（`formats/10-traceability.md`）の
> 正準形＝「**状態 日付 反映先ID**」（例 `済 2026-06-20 QA-007`、未＝`未`、対象外＝`−`）。
> 同梱サンプルは図書貸出ドメイン（`instance/library-lending/docs/10-traceability.md`）。

## 起動方法

`formats/` を fetch するため、静的サーバー経由で開く必要がある（`file://` 直開きは不可）。

リポジトリ（キット）のルートで：

```bash
npx serve . -p 4322
# → http://localhost:4322/dashboard/ を開く
```

サンプル（図書貸出）を見る場合：

```bash
npx serve instance/library-lending -p 4323
# → http://localhost:4323/dashboard/
```

ルートを配信するのは、`/dashboard/` と文書フォルダ（`/formats/` や `/docs/`）の両方が
見える状態にするため（ダッシュボードは manifest.json の `docsDir` を相対参照する）。

## 設定はすべて manifest.json（index.html は編集不要）

`index.html` は**全プロジェクト共通**。タイトル・文書一覧・データ元などプロジェクトごとの差は、
同じフォルダの **`manifest.json`** だけで吸収する。

| manifest.json の項目 | 意味 |
|---|---|
| `title` / `subtitle` / `pageTitle` | 画面上部とタブの表示 |
| `docsDir` | 文書を読むフォルダ（テンプレ＝`../formats/`、実プロジェクト＝`../docs/`） |
| `docs` | 文書一覧（id・file・title・role・axis・group）。1行足せば俯瞰・マトリクス・閲覧に反映される |
| `aliases` | 参照検出に使う文書の呼び名（長い名前を先に） |
| `instance.file` | 進捗タイムラインが読む要件トレーサビリティ文書のパス |

## 進捗タイムラインのデータ

`instance.file` が指す Markdown 内の、ヘッダに `要件ID … 品質保証` を含む表（＝トレーサビリティ・マトリクス §2）を読む。

- 列：`要件ID | タイトル | 情報定義 | 機能仕様 | API仕様 | アーキテクチャ | 実装 | 品質保証`
- セル：`状態 日付 反映先ID` の順・半角スペース区切り（例 `済 2026-06-20 QA-007`）
- 状態は `未`／`反映中`／`済` の3語だけ。対象外の層は `−`
- チケットの代表状態（「実装中」等）は書かない → 表から自動で「〜まで済」と表示される

## 新しいプロジェクトで使う

`index.html` をそのままコピーし、`manifest.json` を書くだけ（`kit-install` スキルが導入時に自動でやる）。
